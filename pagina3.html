GRANT

O comando GRANT é usado para conceder permissões a usuários em um banco de dados. Ele permite definir quem pode executar ações como SELECT, INSERT, UPDATE, DELETE, entre outras.

Você também pode conceder privilégios administrativos com GRANT ALL PRIVILEGES.

DENY

O comando DENY não é suportado nativamente no MySQL. Ele é comum em outros SGBDs como SQL Server, onde é usado para negar explicitamente permissões, mesmo que o usuário tenha herdado permissões por grupo.

No MySQL, para bloquear acesso, você deve evitar o uso de GRANT ou usar REVOKE para remover permissões.

REVOKE

O comando REVOKE é usado para remover permissões previamente concedidas com GRANT. Ele é essencial para manter a segurança e controle de acesso.

Após o REVOKE, o usuário perde os privilégios especificados.

Tipos de Dados MySQL

O MySQL oferece diversos tipos de dados para armazenar informações de forma eficiente:

  • INT – números inteiros
  • VARCHAR(n) – texto com tamanho variável
  • TEXT – texto longo
  • DATE – datas (AAAA-MM-DD)
  • DATETIME – data e hora
  • BOOLEAN – verdadeiro ou falso (representado como TINYINT)
  • FLOAT / DOUBLE – números decimais

Escolher o tipo correto impacta diretamente na performance e integridade dos dados.
